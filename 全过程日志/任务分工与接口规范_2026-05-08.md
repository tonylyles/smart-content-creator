## 二、刘凯睿：内容生成与质量保障

### 你要写什么

| 模块 | 文件 | 当前状态 | 你要做的 |
|------|------|---------|---------|
| 内容生成 | `generator.py` | DeepSeek已通，但输出格式不够规范 | 统一输出结构，确保返回 title + markdown + html + 配图建议 |
| 提示词引擎 | `prompt_engine.py` | 产品知识模板已注入 | 你来维护和优化，产品参数变更找圳刚要最新数据 |
| RAG检索 | `rag/retriever.py` + `rag/vector_db.py` | **返回空值，整条链路断的** | 让知识库有数据、检索能出结果，这是最紧急的 |
| 质量评估 | `evaluator.py` | 代码在，没跑过真实数据 | 跑通一条完整评估流程 |
| 用户界面 | `ui.py` | 只有框架 | Flask页面，最低要能：创建发布计划、查看生成结果、审核通过/打回 |

### 优化方向

1. **配图建议** — 赛题明确要求"包含配图建议"，生成时必须输出图片位置+描述+搜索关键词
2. **微信排版** — 微信编辑器会剥离 `<style>` 和 `class`，只有 `style=""` 内联样式生效。你负责确保输出的HTML在微信里排版正确
3. **内容多样性** — 赛题要求覆盖"技术/事件/政策"三大内容类型，目前只有技术类能跑
4. **RAG是关键瓶颈** — 知识库空=检索空=生成全靠LLM编=内容不靠谱。找圳刚要数据，你负责让检索链路通

### 命名规范

- 你的模块都在 `src/` 和 `src/rag/` 下
- 函数命名：`generate()`, `retrieve()`, `evaluate()` — 已经OK
- 返回值统一用 `dict`，不要混用 `str` 和 `dict`
- `prompt_engine.py` 里的场景配置（`SCENE_CONFIG`）如果新增场景，用小写下划线命名，如 `sludge_drying_solution`

## 三、胡圳刚：数据采集与知识库

### 你要写什么

| 模块 | 文件 | 当前状态 | 你要做的 |
|------|------|---------|---------|
| 行业爬虫 | `spiders/` 目录 | 原爬虫抓通用科技新闻，跟环保无关 | **改成环保行业数据源**，赛题要求覆盖≥5个数据源 |
| 知识库 | `knowledge_base.py` + `data/content.db` | **数据库几乎是空的** | 大量填充吉康环境产品资料、环保政策、行业术语 |
| 数据清洗 | `data_cleaner.py` | 代码在，没跑过 | 让爬虫抓的数据能清洗后入库 |
| 数据存储 | `data_storage.py` | 代码在，未接入流水线 | 对接工作流引擎，让其他模块能读写数据 |

### 优化方向

1. **数据源必须换** — 当前 `news_crawler.py` 抓的是IT之家、36氪，跟吉康环境零关系。改成环保在线、北极星环保、生态环境部等（我写了个 `eco_crawler.py` 作参考，你可以改也可以重写）
2. **知识库是最紧急的事** — 没有数据，凯睿的RAG检索就是空的，整条链路断。至少需要：
   - 吉康环境全部产品信息（型号、参数、案例）
   - 广东省环保政策文件（十四五规划、双碳政策、固废标准）
   - 行业技术术语表
   - 我在 `data/jikang_knowledge.md` 里整理了一些，你在此基础上扩充
3. **爬虫要能跑出真实数据** — 不能只返回空列表。测试方法：直接 `python -m src.spiders.spider_manager` 看结果
4. **数据流要贯通** — 爬虫抓数据 → 清洗 → 入库 → 凯睿的RAG能检索到，这条链路必须通

### 命名规范

- 爬虫相关文件放在 `src/spiders/` 下
- 爬虫类命名：`XxxCrawler`（如 `EcoNewsCrawler`、`PolicyCrawler`）
- 解析器方法命名：`_parse_xxx`（如 `_parse_hbzhan`、`_parse_mee`）
- 数据源配置用常量 `SOURCES`，按 `industry / policy / tech` 分类
- 知识库相关命名：`knowledge_base.py` 已有，别新建文件

## 五、命名规范（全员遵守）

| 类别 | 规范 | 正确示例 | 错误示例 |
|------|------|---------|---------|
| 文件名 | 小写下划线 | `eco_crawler.py` | `EcoCrawler.py` |
| 类名 | 大驼峰 | `ContentGenerator` | `content_generator` |

| 常量 | 大写下划线 | `SCENE_CONFIG` | `sceneConfig` |
| 私有方法 | 前置下划线 | `_parse_html()` | `parseHtml()` |
| 配置key | 小写下划线 | `llm_base_url` | `llmBaseUrl` |
| 目录名 | 小写 | `spiders/`, `rag/` | `Spiders/`, `RAG/` |
| 模块归属注释 | 文件第一行docstring | `"""内容生成引擎 - 刘凯睿负责"""` | 无注释 |
