│  ## **四、 数据清洗与结构化规则**                                                   │                                                                                            
│                                                                                     │                                                                                            
│  清洗流程遵循 **“抽取-清洗-转换-标准化”** 的流水线。                                │                                                                                            
│                                                                                     │                                                                                            
│  ### **4.1 清洗规则**                                                               │                                                                                            
│  1.  **去噪与无关内容移除**：                                                       │                                                                                            
│      *                                                                              │                                                                                            
│  **规则**：删除HTML标签、脚本(`<script>`)、样式(`<style>`)、导航栏、页脚、广告区块  │                                                                                            
│  、版权声明等与正文无关的通用内容。                                                 │                                                                                            
│      *                                                                              │                                                                                            
│  **方法**：在`BeautifulSoup`解析时，通过CSS选择器或标签特征识别并移除。使用正则表   │                                                                                            
│  达式清理残留的`&nbsp;`等HTML实体。                                                 │                                                                                            
│  2.  **正文内容提取与精炼**：                                                       │                                                                                            
│      *   **规则**：识别并保留核心正文区域（通常位于特定`<div>`容器内）。            │                                                                                            
│      *                                                                              │                                                                                            
│  **方法**：基于密度和标签结构算法（如`readability-lxml`或自定义启发式规则）提取正   │                                                                                            
│  文。合并被`<p>`标签分割的连续段落。                                                │                                                                                            
│  3.  **文本格式化**：                                                               │                                                                                            
│      *                                                                              │                                                                                            
│  **规则**：将多个连续空白字符（空格、换行、制表符）替换为单个空格或标准段落分隔。   │                                                                                            
│      *   **方法**：使用`re.sub(r'\s+', ' ', text)`进行清理。                        │                                                                                            
│  4.  **关键元数据抽取**：                                                           │                                                                                            
│      *                                                                              │                                                                                            
│  **规则**：必须提取`标题`、`发布时间`、`来源网站`、`原文链接`。尽可能提取`作者`、`  │                                                                                            
│  关键词/标签`、`文章分类`。                                                         │                                                                                            
│      *   **方法**：                                                                 │                                                                                            
│          *                                                                          │                                                                                            
│  **日期**：从`<meta>`标签（如`article:published_time`）、特定CSS类或正文开头使用正  │                                                                                            
│  则表达式匹配。统一转换为`YYYY-MM-DD HH:MM:SS`格式。                                │                                                                                            
│          *                                                                          │                                                                                            
│  **分类/标签**：从URL路径、页面面包屑导航或特定标签区块解析。若无，则根据正文关键   │                                                                                            
│  词（如“政策”、“技术”、“案例”）进行**自动分类打标**。                               │                                                                                            
│  5.  **去重规则**：                                                                 │                                                                                            
│      *   **基于URL去重**：同一URL视为重复。                                         │                                                                                            
│      *                                                                              │                                                                                            
│  **基于内容相似度去重**：对于不同来源报道同一事件，使用SimHash或TF-IDF计算文本相似  │                                                                                            
│  度，若超过阈值（如0.9），则保留发布时间最早或来源最权威的一条。                    │                                                                                            
│  6.  **企业内部数据特殊处理**：                                                     │                                                                                            
│      *   **规则**：将PDF/Word中的表格数据转化为结构化JSON字段（如`{“model”:         │                                                                                            
│  “JK-GZ-100”, “dewatering_capacity”: “10t/d”, “power”: “35kW”}`）。                 │                                                                                            
│      *                                                                              │                                                                                            
│  **方法**：使用`tabula`（PDF表格）或`pandas.read_html`（HTML表格）提取表格。对非标  │                                                                                            
│  准格式，制定模板规则进行文本解析。                                                 │                                                                                            
│                                                                                     │                                                                                            
│  ### **4.2 数据标准化与输出**                                                       │                                                                                            
│  1.  **统一数据模式**：                                                             │                                                                                            
│      *   所有清洗后的数据必须遵循以下JSON Schema：                                  │                                                                                            
│      ```json                                                                        │                                                                                            
│      {                                                                              │

│        “publish_time”: “字符串，ISO 8601格式时间”，                                 │
